最近在尝试 opencode 这个 AI 编程助手，发现它确实能极大提升开发效率。不过我的电脑是 Windows，而 opencode 在 Linux 环境下用起来更顺手，于是决定在 Windows 上装个 WSL（Windows Subsystem for Linux），把项目搬进去用 opencode。记录一下整个过程，也给同样在 Windows 上挣扎的开发者们一点参考。

## 为什么选 WSL？

以前我都是直接在 Windows 上搞开发，但很多工具链、脚本在 Linux 下更原生，尤其是涉及到服务器部署、Docker 之类的场景。虚拟机太重，双系统又麻烦，WSL2 是个不错的折中方案——既能在 Windows 上用熟悉的软件，又能享受 Linux 的命令行环境。

## 安装 WSL2

微软官方文档已经写得很详细了，但实际走一遍还是会踩几个坑。

1. **启用 WSL 功能**  
   用管理员权限打开 PowerShell，运行：
   ```powershell
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   ```
   重启电脑。

2. **安装 Linux 发行版**  
   重启完成后可以运行以下代码来看目前有哪些版本可以使用
   ```powershell
   # 查看所有已安装的 WSL 发行版及其状态
   wsl --list --online
   ```

3. **升级到 WSL2**  
   在 PowerShell 里执行：
   ```powershell
   wsl --set-default-version 2
   ```
   <div class="note">
   Windows11自带即为WSL2，不需要手动更新
   <div>

4. **再次重启**  
   确保设置生效  

## 配置开发环境

WSL 装好后，第一件事是换源、装常用工具。

```bash
# 备份原来的源
sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak
# 换成阿里云源（国内快很多）
sudo sed -i 's/archive.ubuntu.com/mirrors.aliyun.com/g' /etc/apt/sources.list
sudo apt update && sudo apt upgrade -y

# 装基础工具
sudo apt install -y git curl wget zsh build-essential
```

<div class="note" style="margin-bottom: 1em;">
当然，这不重要，我就没换
</div>

然后配置 Git、SSH 密钥，把本地的项目拷贝到 WSL 里，如果是在GitHub或者Gitee上的项目再在WSL中`mkdir`新建一下项目目录，再到里面克隆一下仓库即可

## 安装 opencode

opencode 是一个命令行 AI 编程助手，需要 Python 环境。WSL 默认带了 Python3，直接用 pip 安装就行。

```bash
pip install opencode
```

不过这里有个小坑：如果系统里同时有 Python2 和 Python3，记得用 `pip3`。另外建议用 `--user` 参数装到用户目录，避免权限问题。

安装完成后，运行 `opencode --help` 确认一切正常。

## 第一次使用 opencode

在项目目录下直接跑 `opencode`，它会启动一个交互式会话。我试着让它帮我修改“关于”页面的样式，效果出乎意料的好。

opencode 不仅能理解代码上下文，还能根据现有代码风格给出修改建议。比如我原来的“关于”页面比较朴素，opencode 建议加入毛玻璃效果、网格布局、动态悬停，并且自动生成了完整的 CSS 代码。最让我惊喜的是，它还能考虑到深色/浅色主题的适配，这在手动写样式时很容易忽略。

## 遇到的坑与解决

1. **文件权限问题**  
   Windows 和 WSL 的文件系统权限不互通，从 `/mnt/c/` 拷贝过来的文件权限可能不对，导致脚本无法执行。用 `chmod +x script.sh` 修复即可。

2. **Git 换行符**  
   Windows 的 CRLF 和 Linux 的 LF 混在一起，Git 会报 warning。在 WSL 里设置：
   ```bash
   git config --global core.autocrlf input
   ```

3. **网络问题**  
   WSL2 的网络和 Windows 主机是分开的，有时候代理配置需要手动同步。在 `~/.bashrc` 里加上：
   ```bash
   export http_proxy=http://127.0.0.1:7890
   export https_proxy=http://127.0.0.1:7890
   ```

4. **性能问题**  
   项目文件放在 Windows 分区（`/mnt/c/`）时，IO 性能会比较差。建议把代码放在 WSL 的家目录（`~/projects/`）里，速度会快很多。

## 总结

折腾了一天，终于能在 Windows 上愉快地使用 opencode 了。WSL2 确实是个好东西，让 Windows 开发者也能无缝享受 Linux 生态。opencode 的表现也很亮眼，不仅代码生成能力强，还能理解项目上下文，给出符合代码风格的建议。

如果你也在用 Windows 开发，不妨试试 WSL + opencode 这个组合。虽然前期配置有点麻烦，但一旦跑起来，效率提升是实实在在的。

**一点小建议**：
- 尽量把代码放在 WSL 分区，避免跨文件系统性能损失。
- 定期备份 WSL 系统（`wsl --export`），免得玩坏了重装。
- opencode 虽然智能，但生成的代码还是要自己 review，别完全依赖它。

接下来我打算用 opencode 继续优化这个博客的其他页面，比如文章列表、首页布局。有机会再和大家分享更多使用心得。
